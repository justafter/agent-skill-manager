# D0 阶段基础设施补齐开发计划

本计划针对 M1/D0 阶段的“基础设施补齐”工作，目标是为 Skill Manager 补齐配置深度合并、注册表读写、安全路径校验（Path Guard）、统一错误类型和诊断命令等核心基础能力。

## 涉及文件

- `src/core/config.ts` - 配置加载、深度合并与保存
- `src/core/registry.ts` - 统一读写 `library/registry.json` (新建)
- `src/projects/guard.ts` - 安全路径越界校验（Path Guard）与路径真实化
- `src/utils/paths.ts` - 跨平台用户目录展开
- `src/utils/fs.ts` - 添加目录/文件可写性检查
- `src/utils/errors.ts` - 统一应用错误类型 `AppError` (新建)
- `src/cli/doctor.ts` - 诊断命令检查路径与可写性
- `package.json` - 检查 `@types/diff` 是否已存在；当前若已添加则不重复处理

## 1. 拟实施工作项

### 1.1 错误模型与基础工具
- 在 `src/utils/errors.ts` 中实现自定义错误 `AppError`，包含 `code`、`message` 和 `details`。
- 在 `src/utils/fs.ts` 中实现 `isWritable(target: string)`。
- 在 `src/utils/paths.ts` 中实现 `expandUserProfile(value: string)` 以展开 `%USERPROFILE%`。

### 1.2 注册表读写
- 创建 `src/core/registry.ts` 并提供 `loadRegistry(root)` 和 `saveRegistry(registry, root)` 函数。
- 确保 `loadRegistry` 健壮：若 `library/registry.json` 不存在则返回默认的空模板。

### 1.3 配置深度合并与保存
- 在 `src/core/config.ts` 中提供 `deepMerge` 实现。
- 更新 `loadConfig`：依次加载基准配置 `skill-manager.config.json` 和用户配置 `%USERPROFILE%\.skill-manager\config.json` 并合并。
- 新增 `saveConfig` 写入用户级配置。
- 若旧实现或旧文档使用 `%USERPROFILE%\.skill-manager.config.json`，在 D0 中统一迁移到目录式路径，后续 PID、日志、配置都落在 `%USERPROFILE%\.skill-manager\` 下。

### 1.4 安全路径校验 (Path Guard)
- 实现安全目录白名单校验：确保一切写操作严格限制在已注册项目、适配器用户目录、备份目录、模板目录和本工具管理的资源目录内。
- 白名单必须是具体路径，不得把整个仓库根目录作为通用允许写入根。允许写入范围包括：
  - `library/skills`
  - `library/registry.json`
  - `backups`
  - `ruleTemplateDir`
  - 已注册项目目录
  - 已启用适配器的用户级 Skill 目录
- 实现 `resolveRealpath` 支持对不存在路径的级联 Realpath 物理化解析，防止软链接逃逸。

### 1.5 诊断命令与编译修复
- 检查 `package.json` 的 `devDependencies` 是否已包含 `@types/diff`；若当前仓库已存在，则只在计划交接中标记为已满足，不再重复修改。
- 升级 `asm doctor` 使其检测 `backupDir`、`ruleTemplateDir`、各适配器目录及所有注册项目的路径存在性与可写性。

## 2. 验证机制

### 自动化测试
- 新建 `tests/integration/infrastructure.test.ts` 进行集成测试。
- 执行 `pnpm run typecheck`。
- 执行 `pnpm run test`。

### 手动检查
- 运行 `pnpm dev doctor` 查看格式化 JSON 的诊断输出。
