# D11 会话记录迁移与恢复开发记录

日期：2026-07-13

状态：核心开发完成，等待真实关闭态验收

## 1. 已完成

- 在 `sessions.archiveDir` 下增加统一归档目录配置，并支持 Claude、Codex、Gemini 的可配置源根。
- 新增独立 `src/sessions` 领域，CLI、Server 和 Web 共用同一套扫描、plan 和 apply 逻辑。
- Claude 按 transcript JSONL 与同 UUID 目录组成 bundle；不修改 `history.jsonl`。
- Codex 扫描 rollout JSONL，优先使用 `session_meta.payload.id` 并关联 `session_index.jsonl`；不修改索引和 SQLite。
- Gemini 只迁移 UUID 目录，区分完整会话和 artifact-only，排除 `tempmediaStorage` 等非 UUID 目录。
- 迁移采用 copy、checksum verify、归档盘内原子提交、删除源；还原目标存在时拒绝覆盖。
- 增加活动状态、稳定窗口、Windows 独占访问探测、空间检查、路径越界保护、manifest 校验和持久化 operation journal。
- 新增 `asm sessions` CLI、`/api/sessions` 路由、Web `/sessions` 双列表页面和确认窗口。
- 参考 cc-switch Session Manager 优化展示：紧凑会话行、相对时间、独立选中态、列表/分类切换、Agent → 项目目录折叠、双列表 + 当前会话详情和响应式布局。
- 新增三种 transcript 的共享只读解析和消息 API；点击会话后按需展示用户、助手、系统和工具消息，归档 payload 也可直接查看。
- 过滤 Claude thinking、Codex reasoning/developer 指令和 Gemini 重复历史/ thinking；增加行数、消息数、单条正文和总正文读取上限。
- 未复制 cc-switch 的永久删除、复制恢复命令或终端恢复业务。
