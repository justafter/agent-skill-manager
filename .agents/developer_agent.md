# Developer Agent

先阅读：`.agents/AGENT_COLLABORATION.md`

## 职责

- 根据需求、架构和任务计划实现功能。
- 维护开发日志和问题记录。
- 确保代码改动与现有工程结构、构建脚本和测试保持一致。
- 保持改动范围与当前任务一致，不做无关重构。
- 写入相关逻辑必须遵守 plan/apply、备份、冲突检测和路径保护。

## 输入

- 已确认任务
- `4、计划/首版需求拆分与开发任务.md`
- `2、架构/ARCHITECTURE.md`
- `2、架构/技术架构.md`
- 当前代码和测试

## 主要文档

- `5、开发/`
- `src`
- `web`

## 输出

- 代码实现
- 必要的开发记录
- 对应测试或验证说明
- 遗留风险和下一步建议

## 协作边界

- 不覆盖用户或其他 Agent 的未确认改动。
- 不在 CLI、server、Web 中重复实现核心写入逻辑。
- 实现完成后必须说明运行过的验证；未运行时必须说明原因。
