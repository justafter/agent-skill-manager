# D8 规则模板同步开发计划

本计划针对 M1/D8 阶段的规则模板同步工作，实现项目级规则文件 `CLAUDE.md` / `AGENTS.md` / `GEMINI.md` 的安全差异比对、双向同步与局部托管块替换。

## 涉及文件

- `src/types/rule.ts` - 规则与模板类型定义
- `src/rules/plan.ts` - 规则同步计划生成核心逻辑（接受可选 `templateDir` 参数；不传则从 `config.ruleTemplateDir` 解析或回退 `<root>/library/rules`）
- `src/rules/apply.ts` - 规则同步计划物理执行与备份（同上：`templateDir` 可选参数；`applyRuleSync` 与 `crossSyncRule` 行为独立）
- `src/rules/template.ts` - 规则模板加载器（`<templateDir>/<agent>/<file>`）
- `src/cli/project.ts` - 扩展 CLI `asm project` 命令行组以接入规则同步
- `src/server/routes/projects.ts` - 增加规则 diff 与同步接口；从 config 解析 `templateDir` 后再调 `planRuleSync` / `applyRuleSync`
- `src/server/routes/rules.ts` - 规则模板库聚合接口（`GET /api/rules`、`GET /api/rules/diff`），模板目录解析交给 `resolveTemplateDir(config)`；为 D6 §2.11 RulesPage 提供数据
- `src/server/routes/rule-template-dir.ts` - **新增**，模板目录 GET / PUT；PUT 自动 mkdir 并写入 user config
- `web/src/pages/ProjectSpacePage.tsx` - 项目空间详情中嵌入规则 Diff 视图与同步操作
- `web/src/pages/RulesPage.tsx` - 独立路由 `/rules` 的 Rule 模板库页面（D6 §2.11；与项目空间页面的"AI 规则同步 (D8)" Tab 共享同一套 `planRuleSync` / `applyRuleSync` 核心）；顶部 toolbar 含"切换模板目录…"按钮
- `web/src/components/DirectoryPicker.tsx` - 通用目录选择器，切换模板目录的 modal 与首页"导入技能"、项目"添加项目"复用同一组件
- `web/src/components/DiffView.tsx` - 规则文本 Patch Diff 渲染器

## 1. 设计原则

- **托管块优先（Block-level replacement）**：
  - 默认为 `block` 模式：仅查找并替换项目规则文件中的 `<!-- BEGIN AgentSkillManager:<agent> --> ... <!-- END AgentSkillManager:<agent> -->` 标识块，保留用户在块外手写的自定义规则，避免覆盖用户的劳动成果。
- **安全拦截与覆盖确认**：
  - 若项目下规则文件已存在，但**没有托管块标识**，则默认判定为冲突（Conflict），静默同步将被拒绝，必须由用户显式授权 `overwrite`（完全覆写）模式。
- **双向拉取能力**：
  - 支持 `pull` 模式，将项目中的规则文件内容反向同步并覆写更新本地的权威模板库（`library/rules/<agent>/...`）。
- **同步前备份**：
  - 在对项目文件执行 `overwrite` 或 `block` 模式写入，或执行 `pull` 反向覆写本地模板前，自动在 `backups/` 目录下创建文件级快照备份。

## 2. 拟实施工作项

### 2.1 规则同步计划核心 (`src/rules/plan.ts`)
实现规则计划生成器 `planRuleSync`，传入项目、适配器代理名，计算：
- **状态评估**：
  - 检查项目内目标文件（例如 `CLAUDE.md`）是否存在。
  - 若不存在，推荐模式为 `create`。
  - 若存在，查找 `<!-- BEGIN AgentSkillManager:<agent> -->` 的起止索引。
  - 若存在该托管块，读取块内文本并与本地规则模板进行哈希或文本比对。若完全一致，推荐模式为 `identical`（无需修改）；若不一致，推荐模式为 `block`。
  - 若托管块不存在，推荐模式为 `conflict`。
- **Diff 生成**：
  - 使用 `diff` 模块生成从“当前项目文件内容”到“预期写入内容”的 Unified Patch（文本补丁差异字符串）。

### 2.2 规则物理应用引擎 (`src/rules/apply.ts`)
实现规则物理写入与拉取器 `applyRuleSync`：
- **Block 写入**：如果文件不存在，则直接创建；如果存在且含有托管块，读取原文件内容，仅替换托管块部分，重新原子写入。
- **Overwrite 写入**：完全覆写目标文件。
- **Pull 写入**：读取项目文件内容，用其覆写更新本地权威库对应的模板文件。
- **备份创建**：在执行上述任何写操作前，自动备份目标文件到 `backups/` 中。

### 2.3 CLI 命令扩展 (`src/cli/project.ts`)
在 `asm project` 命令组中扩充：
- `asm project plan-rules <project-id> --agent <agent>`：扫描并打印规则文件状态、是否存在托管块、并输出 Patch Diff 差异。
- `asm project push-rules <project-id> --agent <agent> [--mode block|overwrite]`：推送规则模板，可指定 `block` 或 `overwrite` 模式。
- `asm project pull-rules <project-id> --agent <agent>`：反向拉取项目中的规则以更新本地模板。

### 2.4 后端路由开发 (`src/server/routes/projects.ts`)
增加以下 API 接口：
- `GET /api/projects/:id/rules/diff`：接收 query 参数 `agent`，返回该项目此 Agent 规则文件的 Diff 信息（包含是否存在、是否含托管块、建议模式、以及 `patch` 差异文本）。
- `POST /api/projects/:id/rules/sync`：接收 body 参数 `{ agent, mode }`，执行物理规则同步，返回应用结果。

### 2.5 前端 Web UI 规则同步对接
- **组件集成与 UI 美化**：
  - 在 `web/src/pages/ProjectSpacePage.tsx` 的项目详情弹窗中，新增一个 **“AI 规则同步”** 卡片栏。
  - 支持切换 Claude (CLAUDE.md)、Codex (AGENTS.md)、Gemini (GEMINI.md) 查看其与本地模板的同步状态。
  - 嵌入 [DiffView.tsx](file:///D:/Project_by_AI/Skill-mamnager/web/src/components/DiffView.tsx) 渲染文本差异，直观展示红绿行的增改细节。
  - 提供 “仅同步托管块 (Block)” 和 “完全覆写 (Overwrite)” 的操作选项及推送、反向拉取按钮。
- **D6 §2.11 Rule 模板库页（独立路由 `/rules`）**：
  - 卡片与 Skill 卡片同源（标题、`<AGENT>` tag、常驻 `本地模板路径` 行、折叠区等宽 tag + 路径）。
  - 每个已注册项目在折叠区里**独立一个子卡片**，每个子卡片三个按钮：查看 Diff / 拉取 ↓ / 推送 ↑；推送 mode 自适应（conflict → overwrite，否则 block）。
  - 服务端 `GET /api/rules` 全量列出所有项目（不按 `enabledAgents` 过滤）；`GET /api/rules/diff` 复用 `planRuleSync`；同步复用 `POST /api/projects/:id/rules/sync`。
  - **不引入用户级 Rule**（与官方约定对齐：Rule 只有项目级文件，没有"全局"安装路径）。
- **§2.6 项目内跨 Agent 互推（D8 扩展 + D6 §2.11）**：
  - **背景**：`library/rules/<agent>/<file>` 是开发目录，operator 在其中任一文件中手写完成后想推到项目；同时 operator 也常需要把项目内**一个 Agent 文件**的内容推到同项目的**另一个 Agent 文件**（典型场景：CLAUDE.md 写得很完整，复用到 AGENTS.md / GEMINI.md）。
  - 新增核心函数 `crossSyncRule(projectId, sourceAgent, targetAgent, mode)`，mode ∈ {`'block'`, `'overwrite'`}。
  - **block 模式**：从源文件提取 `<sourceAgent>` 托管块，重新包成 `<targetAgent>` 托管块写入目标；目标文件保留块外内容。
  - **overwrite 模式**：源文件整文件覆写目标。
  - 写入前自动备份目标文件到 `backups/<id>/project/<file>`；路径安全校验 `assertInsideProject(...)`。
  - **API**：`POST /api/projects/:id/rules/cross-sync { sourceAgent, targetAgent, mode }`；同源同目标 → 400。
  - **UI**：Rule 模板库页底部新增"项目内跨 Agent 互推"区块，按已注册项目分组，每组一张 `3 × 3` 矩阵，行=源、列=目标、对角线禁用，每格 `block` / `overwrite` 两个按钮（带 `window.confirm` 二次确认）。
  - **不接 Watch**：仅手动触发，不监听项目目录变化，不自动 plan / push。

## 3. 验收口径

### 3.1 自动化测试
在 `tests/integration/rules.test.ts` 中完成：
- 测试无规则文件时直接 `create` 创建规则。
- 测试有规则文件且含托管块时，以 `block` 模式推送只替换托管块，保留用户的自定义手写内容。
- 测试无托管块时，阻断 `block` 写入，验证在 `overwrite` 授权下全量覆写。
- 测试 `pull-rules` 反向拉取规则，验证本地模板库被成功安全覆写更新。

### 3.2 手动检查
- 在 Web 端项目空间中点击管理，切换至规则同步，查看 Diff 渲染正确。
- 修改本地模板后，在 UI 上点击“仅同步托管块”，检查项目对应规则文件中的非托管块内容未被破坏。
- 访问 `/rules` → `Rule 模板库` → 展开任一模板卡片 → 检查"本地模板路径"显示 `library/rules/<agent>/<file>`；折叠区列出全部已注册项目的项目级路径，状态为"已存在/未创建"；点击 `查看 Diff` 加载 patch；点击 `推送 ↑` 时 `mode` 自适应（conflict → overwrite）；点击 `拉取 ↓` 后本地模板文件被覆写更新。
- **跨 Agent 互推**（Rule 模板库页底部矩阵）：
  - 源 = 项目内 `CLAUDE.md`、目标 = 同项目 `AGENTS.md`，点击 `block`：源文件 CLAUDE.md 中 `<claude>` 托管块内容被重新包成 `<codex>` 托管块写入 AGENTS.md；目标文件块外内容保留。
  - 模式 = `overwrite`：目标文件被源文件**整文件**覆写；`backups/<id>/project/AGENTS.md` 已生成。
  - 同源同目标 → 按钮 disabled + 后端 400。
  - 源文件缺失 → 后端报 `Source rule file does not exist: ...`。
  - 源文件无托管块 + `block` 模式 → 后端报 `crossSyncRule: source file has no managed block for "<agent>"`。
