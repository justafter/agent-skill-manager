## 3. `library/` 资源库

`library/` 由管理器负责写入、由用户通过 UI/CLI 维护。**这是 Skill 管理器的"唯一真源"**。

### 3.1 `library/skills/`

存放所有已导入的 Skill 源。每个 Skill 是一个标准目录：

```text
library/skills/<skill-name>/
├── SKILL.md           # 必填：YAML frontmatter + Markdown 说明
├── scripts/           # 可选：脚本
├── references/        # 可选：参考文档
└── assets/            # 可选：静态资源
```

> **注意**：用户在外部开发目录（如 `D:\MySkillDevelopment\<skill>`）开发的 Skill，经 `asm import` 引用后，源依然可以留在开发目录；首版在 `library/skills/` 中保存一份直接拷贝，并在注册表中保留原始 `localPath`。符号链接只作为后续扩展，不进入首版目录约定。

### 3.2 `library/rules/`

按 Agent 分类的规则模板：

```text
library/rules/
├── claude/CLAUDE.md
├── codex/AGENTS.md
├── gemini/GEMINI.md
└── common/AIWorkLog.md     # 通用工作日志模板
```

每个文件可包含托管块占位符：

```md
<!-- BEGIN AgentSkillManager:codex -->
... 由管理器维护的 Codex 项目规则 ...
<!-- END AgentSkillManager:codex -->
```

### 3.3 `library/registry.json`

注册表（见 [ARCHITECTURE.md §4.2](../2、架构/ARCHITECTURE.md)）。

## 4. `backups/` 备份目录

按 `backupId`（如 `20260703-153000`）分目录：

```text
backups/
└── 20260703-153000/
    ├── registry-snapshot.json    # 同步前的注册表快照
    └── <project-or-scope>/
        ├── claude/
        │   └── <skill-name>/     # 原 Skill 目录的完整拷贝
        └── AGENTS.md             # 原规则文件
```

每次写操作前由 `src/backup/` 自动生成；`asm restore <backupId>` 时按索引还原。

## 5. `src/` 源代码

### 5.1 `src/cli/` - 命令行

基于 `commander` 实现的 CLI。`asm` 为可执行名。

| 子命令 | 入口文件 | 说明 |
| --- | --- | --- |
| `asm list` | `list.ts` | 列出已注册 Skill |
| `asm scan` | `scan.ts` | 扫描 dev / registry / targets |
| `asm import <path>` | `import.ts` | 导入本地 Skill |
| `asm sync` | `sync.ts` | 同步（支持 dry-run） |
| `asm watch` | `watch.ts` | 启动 Watch Mode |
| `asm diff` | `diff.ts` | 文本级 diff |
| `asm backup` | `backup.ts` | 手动备份 |
| `asm restore` | `restore.ts` | 恢复 |
| `asm project` | `project.ts` | 项目子命令组（list / add / scan / inject / plan-rules / push-rules / pull-rules） |
| `asm doctor` | `doctor.ts` | 环境自检 |

### 5.2 `src/server/` - 本地后端

基于 `express` 的 HTTP API，端口默认 `47821`（可在配置修改）。所有写接口遵循 `plan/apply` 协议。

| 文件 | 职责 |
| --- | --- |
| `index.ts` | server 启动入口 |
| `app.ts` | Express app 装配与路由挂载 |

| 路由前缀 | 模块 |
| --- | --- |
| `/api/health` | `routes/health.ts` |
| `/api/config` | `routes/config.ts` |
| `/api/skills` | `routes/skills.ts` |
| `/api/scan` | `routes/scan.ts` |
| `/api/import` | `routes/import.ts` |
| `/api/sync` | `routes/sync.ts` |
| `/api/diff` | `routes/diff.ts` |
| `/api/backups` | `routes/backup.ts` |
| `/api/restore` | `routes/restore.ts` |
| `/api/watch` | `routes/watch.ts` |
| `/api/projects` | `routes/projects.ts` |

### 5.3 `src/core/` - 核心引擎

| 文件 | 职责 |
| --- | --- |
| `engine.ts` | 引擎入口：plan/apply 调度 |
| `plan.ts` | `Plan` / `PlanItem` / `PlanId` 类型 |
| `apply.ts` | `applyPlan(planId)` 主体流程 |
| `config.ts` | 加载并校验 `skill-manager.config.json`，生成 `ResolvedConfig` |
| `fingerprint.ts` | 生成 source/target fingerprint（按相对路径 + sha256） |
| `conflict.ts` | 冲突识别与策略选择 |
| `state.ts` | 内存态 plan 注册表（planId → Plan + TTL） |

### 5.4 `src/adapters/` - Agent 适配器

| 文件 | 目标 |
| --- | --- |
| `types.ts` | `Adapter` 接口定义 |
| `base.ts` | 抽象基类：通用 fingerprint、复制、备份钩子 |
| `claude.ts` | Claude Code 适配器 |
| `codex.ts` | Codex 适配器 |
| `gemini-antigravity.ts` | Gemini / Antigravity 适配器 |
| `registry.ts` | 适配器注册与选择（按 `agent` 名取实现） |

### 5.5 `src/projects/` - 项目工作区

| 文件 | 职责 |
| --- | --- |
| `registry.ts` | `config.projects` 的增删改查 |
| `scanner.ts` | 扫描项目内 `.claude/skills` / `.agents/skills` / `CLAUDE.md` / `AGENTS.md` / `GEMINI.md` 等 |
| `inject.ts` | 项目级 Skill 注入（plan + apply） |
| `guard.ts` | 路径安全校验：写入路径必须落在 `project.path` 内部 |

### 5.6 `src/rules/` - 规则模板同步

| 文件 | 职责 |
| --- | --- |
| `template.ts` | 模板加载、解析、占位符渲染 |
| `block.ts` | 托管块定位、替换、提取 |
| `apply.ts` | `overwrite` / `block` / `pull-template` 三种模式 |
| `diff.ts` | 规则文件文本 diff |

### 5.7 `src/sync/` - 同步引擎

| 文件 | 职责 |
| --- | --- |
| `engine.ts` | 高层 sync API：scan → diff → backup → copy → tag |
| `copy.ts` | 完整目录复制（`fs.cp`） |
| `tag.ts` | 写入/读取 `.skill-manager-deploy.json` |
| `multidirection.ts` | 多端扫描、最新源判定、pull/push 路由 |

### 5.8 `src/backup/` - 备份恢复

| 文件 | 职责 |
| --- | --- |
| `index.ts` | 备份索引 CRUD |
| `create.ts` | 创建一次备份（生成 backupId + 写 snapshot） |
| `restore.ts` | 按 backupId 还原 |
| `cleanup.ts` | 备份清理策略（按保留数 / 时间） |

### 5.9 `src/validation/` - 解析与校验

| 文件 | 职责 |

| `mtime.ts` | 最后修改时间收集 |

| `project.ts` | `Project`、`ProjectConfig` |

| `backup.ts` | `BackupItem`、`BackupIndex` |

### 5.11 `src/utils/` - 通用工具

| 文件 | 职责 |
| --- | --- |

| 目录 | 职责 |
| --- | --- |
