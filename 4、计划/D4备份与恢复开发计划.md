# D4 备份与恢复开发计划

本计划针对 M1/D4 阶段的“备份与恢复”工作，目标是实现管理器内的“备份与恢复”闭环机制，支持手动备份、列表查看以及通过备份 ID 原子回滚恢复本地代码和注册表快照。

## 涉及文件

- `src/backup/create.ts` - 实现手动备份 `createManualBackup` 与扫描获取备份列表 `listBackups`
- `src/backup/restore.ts` - 实现备份还原 `restoreBackup`
- `src/cli/backup.ts` - 实现 `asm backup` 创建/列表命令行
- `src/cli/restore.ts` - 实现 `asm restore <id>` 还原命令行
- `src/server/routes/backup.ts` - 路由适配 API 列表与创建
- `src/server/routes/restore.ts` - 路由适配 API 还原触发

## 1. 拟实施工作项

### 1.1 手动快照备份与列表拉取
- 在 `src/backup/create.ts` 中：
  - 实现 `createManualBackup(root, skillName?, reason?)`：
    - 生成 `bk_<timestamp>_<uuid8>` 格式的 `backupId`。
    - 将 `library/registry.json` 保存到 `registry-snapshot.json` 快照中。
    - 若指定 `skillName`：仅复制并备份 `library/skills/<skill-name>` 目录。
    - 若未指定 `skillName`：复制并备份整个 `library/skills` 目录。
    - 写出 `index.json` 索引文件。
  - 实现 `listBackups(root)`：
    - 遍历 `config.backupDir` 中所有包含 `index.json` 的 `bk_*` 目录。
    - 解析索引元数据，按创建时间倒序（降序）返回数组。

### 1.2 倒序原子恢复机制
- 在 `src/backup/restore.ts` 中实现 `restoreBackup(backupId, root)`：
  - 根据 `backupId` 载入备份 `index.json`。
  - 倒序（Reverse）遍历备份项目列表（优先恢复文件，最后恢复 registry 注册表文件，保障原子事务安全性）。
  - 对 Skill 目录还原前，先物理彻底清空（删除）本地当前目录以防残留，再复制还原备份副本。

### 1.3 命令行接口对接
- 扩展 `src/cli/backup.ts`：
  - 支持 `asm backup` 创建全局手动备份。
  - 支持 `-s, --skill <name>` 参数备份特定技能。
  - 支持 `-l, --list` 表格化展示所有可用备份的详细列表。
- 扩展 `src/cli/restore.ts`：
  - 支持 `asm restore <backup-id>` 触发还原流程并打印详情。

### 1.4 API 路由更新
- 完善 `src/server/routes/backup.ts` 与 `src/server/routes/restore.ts` 以提供对应 HTTP endpoints。
