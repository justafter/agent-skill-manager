# D11 会话记录迁移与恢复开发计划

日期：2026-07-13

状态：Review（核心开发与自动化验证完成，待真实关闭态迁移/还原验收）

## 1. 目标

为 Agent Skill Manager 增加独立的“会话管理”模块，安全管理 Claude Code、Codex、Gemini / Antigravity 的本机会话记录：

- 扫描 Agent 原目录和外部归档目录。
- 展示会话名称、项目、时间、大小、活动状态和完整性状态。
- 对选中会话先生成 plan，再执行迁移或还原。
- 跨盘迁移采用“复制、校验、提交、删除”事务，不直接假设 rename 可以跨盘工作。
- 中断或提交后清理失败时，通过持久化 operation journal 识别 `cleanupPending` 等未完成状态。

## 2. 首版范围

### 2.1 包含

- 用户级配置 `sessions.archiveDir` 与三个 Agent 的源根目录。
- Claude `~/.claude/projects/<project>/<sessionId>.jsonl` 及同名配套目录。
- Codex `~/.codex/sessions/**/*.jsonl`、`~/.codex/archived_sessions/*.jsonl`。
- Gemini `~/.gemini/antigravity/brain/<uuid>/` 会话目录。
- CLI、Server API、Web UI 共用 `src/sessions` 核心逻辑。
- manifest、checksum、空间检查、冲突检查、活动状态检查、路径保护和操作日志。
- 双向迁移与还原，以及批量操作逐项结果。

### 2.2 明确不包含

- 不修改 Codex `session_index.jsonl`、`logs_2.sqlite`、`state_*.sqlite` 或其他数据库。
- 不清空 Claude `history.jsonl`，不迁移 Claude cache、file-history、paste-cache。
- 不迁移 Gemini `tempmediaStorage` 等非 UUID 目录。
- 不合并 JSONL，不覆盖还原目标，不提供“强制覆盖”选项。
- 不承诺归档期间仍可从原 Agent 的历史列表打开会话；还原后才验证 Agent 可重新读取。
- 首版不自动删除中断遗留 staging 或自动重试 `cleanupPending`；操作日志保留现场，后续通过恢复向导处理。

## 3. 当前本机基线

只读勘测基于：Codex CLI 0.139.0、Claude Code 2.1.168、Gemini CLI 0.38.2。

- Claude：`projects` 才是主要会话数据；单个会话可能由 JSONL 和同 UUID 配套目录共同组成。
- Codex：`session_index.jsonl` 当前只有 `id`、`thread_name`、`updated_at`，需要扫描 rollout 并按 ID 关联。
- Gemini：UUID 目录中既有完整 transcript 会话，也有仅 artifacts 的目录；非 UUID 目录必须排除。

该基线只用于适配器 fixture，运行时仍需以目录签名和实际文件结构为准。

## 5. 配置设计

运行时合并配置新增：

```json
{
  "sessions": {
    "archiveDir": "D:\\AgentSessionArchive",
    "agents": {
      "claude": {
        "enabled": true,
        "root": "%USERPROFILE%/.claude"
      },
      "codex": {
        "enabled": true,
        "root": "%USERPROFILE%/.codex"
      },
      "gemini": {
        "enabled": true,
        "root": "%USERPROFILE%/.gemini/antigravity/brain"
      }
    }
  }
}
```

- 用户选择的归档目录只写 `%USERPROFILE%/.skill-manager/config.json`。
- 默认配置中的 `archiveDir` 为 `D:\AgentSessionArchive`；用户级配置中的自定义值（包括显式清空）优先。默认目录不在启动或扫描阶段创建。
- 归档根不能位于任一源根内部，源根也不能位于归档根内部。
- 同盘可用，但 UI 必须提示不会释放系统盘空间。

## 6. 数据契约

### 6.1 SessionRecord

```ts
interface SessionRecord {
  id: string
  agent: 'claude' | 'codex' | 'gemini'
  location: 'agent' | 'archive'
  kind: 'transcript' | 'session-bundle' | 'artifact-only'
  title?: string
  workspacePath?: string
  createdAt?: string
  updatedAt: string
  sizeBytes: number
  fileCount: number
  originalRelativePath: string
  activity: 'idle' | 'busy' | 'unknown'
  integrity: 'unchecked' | 'valid' | 'invalid'
}
```

### 6.2 ArchiveManifest

manifest 至少包含：

- `schemaVersion`
- `archiveId`
- `agent`、`sessionId`、`kind`
- `originalRoot`、`originalRelativePath`、`originalPath`
- `payloadRelativePath`
- `fileCount`、`sizeBytes`、`checksum`
- `createdAt`、`updatedAt`、`archivedAt`
- `adapterVersion`

恢复时只使用“当前配置源根 + originalRelativePath”计算目标，`originalPath` 只用于审计展示。

### 6.3 归档目录

```text
<archiveDir>/
├── .agent-skill-manager/
│   └── operations/<operationId>.json
└── sessions/
    ├── claude/<sessionId>/
    │   ├── manifest.json
    │   └── payload/
    ├── codex/<sessionId>/
    │   ├── manifest.json
    │   └── payload/
    └── gemini/<sessionId>/
        ├── manifest.json
        └── payload/
```

每个 bundle 的 manifest 是权威索引；全局缓存索引损坏时必须能通过扫描 manifest 重建。

## 7. Agent 适配策略

### 7.1 Claude

- 扫描 `projects/*/*.jsonl`，UUID 文件名作为 sessionId。
- 同目录存在 `<sessionId>/` 时，将 JSONL 与配套目录视为一个不可拆分 bundle。
- 只读 `history.jsonl`，按 sessionId 补充标题、项目和时间，不移动、不截断。
- 读取 `sessions/*.json` 活动标记，匹配中的 sessionId 标记为 busy。

### 7.2 Codex

- 扫描 `sessions/**/*.jsonl` 和 `archived_sessions/*.jsonl`。
- 从 `session_meta` 或文件名提取 sessionId、cwd 和创建时间。
- 读取 `session_index.jsonl`，按 id 补充 thread_name 和 updated_at。
- 索引和数据库保持只读。

### 7.3 Gemini

- 只扫描合法 UUID 子目录。
- 整个 UUID 目录为一个 bundle。
- 存在 `.system_generated/logs/transcript*.jsonl` 时标记为 session-bundle。
- 没有 transcript 时标记 artifact-only，并在迁移 plan 中加入 warning。
- 非 UUID 目录默认忽略。

## 8. 迁移与还原事务

### 8.1 Migrate

1. plan 记录源路径、相对路径、size、mtime、checksum 和目标路径。
2. apply 重新校验 plan 快照、活动状态、目标冲突和可用空间。
3. 复制到 `<archiveDir>/.staging/<operationId>/`。
4. 比对源与 staging 的文件数、总字节数和 SHA-256。
5. 在归档盘内原子 rename 到正式 bundle。
6. 写入 manifest，并把 operation journal 标为 committed。
7. 再次校验源未变化后删除源。
8. 删除成功标记 completed；删除失败标记 cleanupPending，保留双份数据。

### 8.2 Restore

1. 校验 manifest schema、checksum 和安全相对路径。
2. 目标存在时直接 conflict，首版不覆盖、不合并。
3. 复制到目标父目录的隐藏 staging 路径。
4. 校验后原子 rename 到最终原路径。
5. 再删除归档 bundle；删除失败标记 cleanupPending。

### 8.3 批量操作

- apply 按项串行执行，避免同一 Agent 并发写入。
- 每项独立提交；失败时返回已完成、失败、未开始列表。
- 不把批量操作描述成全局原子事务。

## 9. API 与 CLI

API：

- `GET /api/sessions`
- `GET /api/sessions/stats`
- `GET /api/sessions/:agent/:location/:sessionId/messages`
- `PUT /api/sessions/config`
- `POST /api/sessions/migrate/plan`
- `POST /api/sessions/migrate/apply`
- `POST /api/sessions/restore/plan`
- `POST /api/sessions/restore/apply`
- `GET /api/sessions/operations`

CLI：

- `asm sessions scan [--agent <agent>]`
- `asm sessions config --archive-dir <absolute-path>`
- `asm sessions migrate --agent <agent> --ids <ids> [--apply]`
- `asm sessions restore --agent <agent> --ids <ids> [--apply]`

不带 `--apply` 时仅输出 dry-run；带 `--apply` 时先生成并展示新 plan，再在同一进程内执行，避免把已过期的内存 plan 跨进程复用。

## 10. Web UI

- 新增 `/sessions` 独立页面和全局导航项“会话管理”。
- Claude / Codex / Gemini Tabs。
- Agent 目录和归档目录双列表，分别选择、过滤和独立滚动；宽屏增加当前会话详情栏。
- 支持“列表 / 分类”视图切换；分类视图按 Agent → 项目目录折叠展示并标注数量，搜索命中时自动展开。
- 点击会话后按需读取 Agent 目录或归档 payload 中的 transcript，并在详情栏展示用户、助手、系统和工具消息。
- 顶部显示源目录大小、归档 payload 大小和迁出逻辑字节数。
- 列表状态包括 idle、busy、unknown、artifact-only、invalid、conflict；apply 结果会显示 cleanupPending。
- 迁移和还原使用独立确认窗口，显示源、目标、大小、警告、冲突和 planId。
- 展示层参考 cc-switch 的紧凑列表、列表/分类切换、相对时间、选中态和会话详情，不引入其永久删除/终端恢复业务。

## 12. 验收标准

- dry-run 不创建目录、不复制、不删除。
- 归档副本未完成校验前绝不删除源。
- 任意失败状态至少保留一份完整有效数据。
- 源在 plan 后变化时 apply 拒绝执行。
- 活动会话或无法确认空闲的会话默认拒绝迁移。
- 归档或恢复目标已存在时拒绝覆盖。
- manifest 被篡改为越界路径时恢复失败。
- Claude 配套目录不被遗漏。
- Codex 索引和 SQLite 在迁移前后字节不变。
- Gemini 非 UUID 目录不进入列表。
- 三种 Agent 的用户/助手消息可从 Agent 目录与归档目录按需查看，内部 thinking/reasoning/developer 指令不进入展示结果。
- 列表/分类视图可切换；分类视图按项目目录分组且搜索结果不会被折叠隐藏。
- 还原后的文件或目录 checksum 与迁移前一致。
- 三个 Agent 各完成一次真实旧会话的迁移、还原和重新打开验证。

## 13. 必测场景

- 临时目录中的同盘迁移与模拟跨盘 fallback。
- 空间不足、归档盘离线、复制中断、checksum 不一致。
- 源内容在 plan 后或复制期间发生变化。
- Windows 路径大小写、长路径、中文路径、junction 越界。
- 目标冲突、manifest 路径穿越、损坏 manifest。
- committed 后源删除失败，operation journal 保留 cleanupPending 状态。
- 批量操作部分成功。
- Claude JSONL + 同名目录 bundle。
- Codex rollout 与无 path 索引关联。
- Gemini 完整会话、artifact-only、非 UUID 目录三种分类。

待验收或后续：

- 尚未配置用户真实归档目录，也未移动任何真实 Agent 文件。
- 需要关闭对应 Agent 后，各选择一条旧会话完成迁移、还原并验证 Agent 能重新打开。
- 自动清理中断 staging 与 `cleanupPending` 恢复向导不在首版自动执行范围内。
