# D8 规则模板同步开发计划

本计划针对 M1/D8 阶段的规则模板同步工作，实现项目级规则文件 `CLAUDE.md` / `AGENTS.md` / `GEMINI.md` 的安全差异比对、双向同步与局部托管块替换。

## 涉及文件

- `src/types/rule.ts` - 规则与模板类型定义
- `src/rules/plan.ts` - [NEW] 规则同步计划生成核心逻辑
- `src/rules/apply.ts` - 规则同步计划物理执行与备份
- `src/cli/project.ts` - 扩展 CLI `asm project` 命令行组以接入规则同步
- `src/server/routes/projects.ts` - 增加规则 diff 与同步接口
- `web/src/pages/ProjectSpacePage.tsx` - 项目空间详情中嵌入规则 Diff 视图与同步操作
- `web/src/components/DiffView.tsx` - 规则文本 Patch Diff 渲染器

## 1. 设计原则

- **托管块优先（Block-level replacement）**：
  - 默认为 `block` 模式：仅查找并替换项目规则文件中的 `<!-- BEGIN AgentSkillManager:<agent> --> ... <!-- END AgentSkillManager:<agent> -->` 标识块，保留用户在块外手写的自定义规则，避免覆盖用户的劳动成果。
- **安全拦截与覆盖确认**：
  - 若项目下规则文件已存在，但**没有托管块标识**，则默认判定为冲突（Conflict），静默同步将被拒绝，必须由用户显式授权 `overwrite`（完全覆写）模式。
- **双向拉取能力**：
  - 支持 `pull` 模式，将项目中的规则文件内容反向同步并覆写更新本地的权威模板库（`library/rules/<agent>/...`）。
- **同步前备份**：
  - 在对项目文件执行 `overwrite` 或 `block` 模式写入，或执行 `pull` 反向覆写本地模板前，自动在 `backups/` 目录下创建文件级快照备份。

## 2. 拟实施工作项

### 2.1 规则同步计划核心 (`src/rules/plan.ts`)
实现规则计划生成器 `planRuleSync`，传入项目、适配器代理名，计算：
- **状态评估**：
  - 检查项目内目标文件（例如 `CLAUDE.md`）是否存在。
  - 若不存在，推荐模式为 `create`。
  - 若存在，查找 `<!-- BEGIN AgentSkillManager:<agent> -->` 的起止索引。
  - 若存在该托管块，读取块内文本并与本地规则模板进行哈希或文本比对。若完全一致，推荐模式为 `identical`（无需修改）；若不一致，推荐模式为 `block`。
  - 若托管块不存在，推荐模式为 `conflict`。
- **Diff 生成**：
  - 使用 `diff` 模块生成从“当前项目文件内容”到“预期写入内容”的 Unified Patch（文本补丁差异字符串）。

### 2.2 规则物理应用引擎 (`src/rules/apply.ts`)
实现规则物理写入与拉取器 `applyRuleSync`：
- **Block 写入**：如果文件不存在，则直接创建；如果存在且含有托管块，读取原文件内容，仅替换托管块部分，重新原子写入。
- **Overwrite 写入**：完全覆写目标文件。
- **Pull 写入**：读取项目文件内容，用其覆写更新本地权威库对应的模板文件。
- **备份创建**：在执行上述任何写操作前，自动备份目标文件到 `backups/` 中。

### 2.3 CLI 命令扩展 (`src/cli/project.ts`)
在 `asm project` 命令组中扩充：
- `asm project plan-rules <project-id> --agent <agent>`：扫描并打印规则文件状态、是否存在托管块、并输出 Patch Diff 差异。
- `asm project push-rules <project-id> --agent <agent> [--mode block|overwrite]`：推送规则模板，可指定 `block` 或 `overwrite` 模式。
- `asm project pull-rules <project-id> --agent <agent>`：反向拉取项目中的规则以更新本地模板。

### 2.4 后端路由开发 (`src/server/routes/projects.ts`)
增加以下 API 接口：
- `GET /api/projects/:id/rules/diff`：接收 query 参数 `agent`，返回该项目此 Agent 规则文件的 Diff 信息（包含是否存在、是否含托管块、建议模式、以及 `patch` 差异文本）。
- `POST /api/projects/:id/rules/sync`：接收 body 参数 `{ agent, mode }`，执行物理规则同步，返回应用结果。

### 2.5 前端 Web UI 规则同步对接
- **组件集成与 UI 美化**：
  - 在 `web/src/pages/ProjectSpacePage.tsx` 的项目详情弹窗中，新增一个 **“AI 规则同步”** 卡片栏。
  - 支持切换 Claude (CLAUDE.md)、Codex (AGENTS.md)、Gemini (GEMINI.md) 查看其与本地模板的同步状态。
  - 嵌入 [DiffView.tsx](file:///D:/Project_by_AI/Skill-mamnager/web/src/components/DiffView.tsx) 渲染文本差异，直观展示红绿行的增改细节。
  - 提供 “仅同步托管块 (Block)” 和 “完全覆写 (Overwrite)” 的操作选项及推送、反向拉取按钮。

## 3. 验收口径

### 3.1 自动化测试
在 `tests/integration/rules.test.ts` 中完成：
- 测试无规则文件时直接 `create` 创建规则。
- 测试有规则文件且含托管块时，以 `block` 模式推送只替换托管块，保留用户的自定义手写内容。
- 测试无托管块时，阻断 `block` 写入，验证在 `overwrite` 授权下全量覆写。
- 测试 `pull-rules` 反向拉取规则，验证本地模板库被成功安全覆写更新。

### 3.2 手动检查
- 在 Web 端项目空间中点击管理，切换至规则同步，查看 Diff 渲染正确。
- 修改本地模板后，在 UI 上点击“仅同步托管块”，检查项目对应规则文件中的非托管块内容未被破坏。
